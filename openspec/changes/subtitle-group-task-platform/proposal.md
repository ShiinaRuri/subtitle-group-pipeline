# Proposal

## Why

字幕组当前依赖聊天工具和零散文件共享推进片源、时轴、翻译、后期、压制和监制协作，项目流转、任务责任和文件交付都缺少统一约束。现在需要把既定规划落成可执行规范，建立一套以项目为中心、支持模板开项、文件版本管理、通知投递和 ASS 专项处理的协作平台规格。

## What Changes

- 新增按项目和按季项目单元组织的字幕组协作平台规范，覆盖片源、时轴、翻译、后期、压制、监制、发布岗位任务链。翻译支持竞争式自选区间领取，片源上传时填写集长度，监制配置每人最大接取长度。
- 新增项目模板与快速开项能力，允许预设岗位、依赖、人员槽位、上传策略、通知策略、ASS 处理策略、成品配置和交付清单。空槽位由监制手动指派或配置为开放领取。
- 新增项目成员主动进组、任务公开接取、任务指派、审核和依赖解锁规则。流水线关键节点（翻译、后期、压制、发布）提交后均需监制审核通过才能进入下一阶段。去掉任务优先级字段，非翻译岗位严格串行执行。
- 新增项目文件筒、文件上传策略矩阵和内置文件版本管理，支持本地存储与 S3 兼容存储。二进制文件实体以每次上传为单位独立创建，同名文件不自动合并。网盘链接类交付物使用独立历史记录模型，不进入二进制文件版本链。文件下载采用临时链接（S3 预签名或本地数据库临时链接），支持敏感标签控制访问范围。
- 新增项目归档、软删除与回收站机制。归档项目任务与文件只读但允许复盘评论，支持取消归档恢复为活跃项目。归档满管理员配置天数后自动清理旧版本，仅保留终稿。项目删除需先归档，删除后进入回收站，超期后物理清理。
- 新增数据保留与清理策略配置，支持管理员设置归档版本清理天数和回收站保留天数。
- 新增基于邮件和 QQ NoneBot 的流程通知能力，QQ 渠道采用群消息加 @ 提醒对应人员，覆盖任务推进、审核、进组审批、文件处理和失败重试。支持渠道升级策略：站内未读则升级邮件，邮件未读则升级 QQ。任务改派时向原被指派人发送转交通知。超期任务自动标记并发送升级通知。
- 新增可配置的用户注册策略，支持禁止注册、允许直接注册、允许注册但需要 QQ 群指令验证三种模式，且验证码在验证前永不过期、验证成功后清除关联关系。注册用户可自选资格标签，管理员考核后手动授予。
- 新增面向 ASS 字幕的服务端解析、多人分段翻译合并、时间轴去重和在线去重页面规范，并要求从第一版开始支持版本间对比、时间轴重叠可视化和冲突解决。去重页面仅对监制和指定角色开放编辑。任务评论区支持引用已有文件版本和行级评论。
- 新增项目 wiki 页面，用于维护项目专有名词和协作说明，支持 Markdown 形式和表格形式编辑，支持审批流配置。
- 新增项目动态时间线（项目级 + 个人全局级）、全局公告和项目公告、三级工作负载视图（个人/项目/全局）。
- 新增项目级搜索与过滤能力。项目/任务支持按名称模糊查询，文件筒支持按文件名和标签过滤，首期不引入全文搜索引擎。
- 新增最终交付清单和成品配置机制，由项目模板定义默认值，项目级监制可覆盖编辑。
- 新增用户资料支持头像上传和昵称修改，头像通过统一存储适配层上传到系统配置的默认存储后端。
- 新增多存储后端配置：管理员可配置多个 S3 或本地存储实例，每个实例可设置最大可用空间配额。项目创建时由监制手动选择存储后端，项目文件全部路由到绑定的后端。

## Capabilities

### New Capabilities

- `project-template-workflow`: 项目模板、按季开项、岗位任务链（片源→时轴→翻译→后期→压制→发布，翻译支持竞争式自选区间领取）、成员进组、任务接取与审核流转。任务可取消（未开始的下游自动冻结，已开始的不受影响但警告），被指派的成员可主动退回任务。去掉任务优先级字段，非翻译岗位严格串行。
- `project-file-bucket-versioning`: 项目文件筒、上传类型策略、本地或 S3 存储适配、二进制文件版本链和历史查看、网盘链接独立历史记录模型。文件实体以每次上传为单位独立创建，同名不自动合并。版本指针采用自动推算语义。文件下载采用临时链接（S3 预签名或本地数据库链接），支持敏感标签控制访问范围。
- `subtitle-merge-dedup`: ASS 解析、竞争式分段翻译领取、服务端合并、时间轴去重、版本间对比和冲突处理。翻译稿直接入文件版本链，合并结果作为独立文件实体。在线去重页面仅对监制和指定角色开放编辑。任务评论区支持引用已有文件版本和 ASS 行级评论。
- `workflow-notification-delivery`: 邮件与 QQ NoneBot 通知、QQ群 @ 提醒、定向投递、失败重试和送达记录。支持渠道升级策略（站内 → 邮件 → QQ）。超期任务自动标记并发送升级通知。任务改派时向原被指派人发送转交通知。任务评论默认静默，仅 @ 时触发定向通知。
- `registration-verification-gating`: 管理员可配置的注册开关、QQ群指令验证、永不过期验证码、待批准账号状态和登录提示。注册用户可自选资格标签，管理员考核后手动授予，服务端维护"申请中 / 已授予"标签状态。
- `project-archive-lifecycle`: 项目归档（只读但允许复盘评论，普通成员不可见）、取消归档恢复、先归档后删除、软删除与回收站、回收站超期物理清理。
- `data-retention-cleanup`: 归档项目旧版本自动清理策略（满 N 天后只保留终稿）、回收站保留天数配置、归档清理与删除的协调规则（删除时跳过待执行的版本清理）。
- `project-search-filter`: 项目/任务按名称模糊查询、文件筒按文件名和标签过滤、首期不引入全文搜索引擎。
- `project-wiki-glossary`: 项目 wiki 页面、专有名词维护、Markdown 编辑和表格编辑，支持审批流配置（管理员全局配置，监制可项目级覆盖）。
- `delivery-checklist`: 最终交付清单由模板继承默认项、项目级监制可编辑、结项时按清单校验。
- `workflow-review-gates`: 流水线审核关卡，翻译/后期/压制/发布提交后均需监制审核通过才能进入下一阶段。
- `project-announcements`: 全局公告（组管理员发布，全员可见）和项目公告（监制发布，项目成员可见）。
- `project-activity-timeline`: 项目级动态时间线（单项目事件流）和个人全局动态时间线（参与项目聚合）。
- `workload-dashboard`: 三级工作负载视图（个人看板 / 监制项目级看板 / 管理员全局看板）。
- `user-profile`: 用户资料页，支持头像上传和昵称修改。头像通过统一存储适配层上传到系统默认存储后端。
- `multi-storage-backend`: 多存储后端实例管理，管理员可配置多个 S3 或本地存储实例及空间配额。项目创建时由监制选择绑定存储后端，项目文件路由到绑定后端。

### Modified Capabilities

无。

## Impact

- 前端需要新增项目模板、项目看板（个人/项目/全局三级）、项目文件筒、历史版本面板、在线去重页面、项目 wiki 页面、公告管理页面、项目动态时间线页面和通知设置页面。
- 前端需要新增注册提示页、待批准登录提示、快捷复制验证命令、技能档案与资格标签申请页面、用户资料页（头像上传与昵称修改）、管理员后台的注册策略配置页面和存储后端配置页面。
- 后端需要新增项目、任务、文件、通知、字幕处理、认证注册校验、归档生命周期、数据保留清理、用户资料和存储后端管理等模块及对应 REST API。
- 数据模型需要新增项目模板、按季项目单元、任务依赖、文件版本、上传策略、通知送达、翻译提交、冲突处理、注册策略、注册验证记录、项目 wiki 文档、网盘链接历史、审核快照、交付清单、标签申请、公告、归档状态、回收站、存储后端实例和项目存储绑定等表结构。
- 文件存储需要支持多实例本地磁盘和多 S3 兼容对象存储配置，每个实例可设置空间配额；项目创建时绑定具体存储后端，二进制文件禁止静默覆盖已有版本，网盘链接使用独立历史记录模型。
- 通知与注册验证基础设施需要接入 SMTP 和 QQ NoneBot HTTP 接口，并监听指定 QQ 群的校验指令与群消息 @ 投递。
