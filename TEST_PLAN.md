# SubtitleSync 端到端浏览器测试计划

基于 `openspec/changes/subtitle-group-task-platform` 功能文档的全流程浏览器测试计划。所有测试步骤均基于真实浏览器操作（页面导航、点击、输入、表单提交）。

---

## 测试环境

| 项 | 值 |
|---|---|
| 前端地址 | http://localhost:5173 |
| 后端地址 | http://localhost:3000/api/v1 |
| 浏览器 | Chromium/Edge（Playwright） |
| 默认管理员 | admin / admin123 |
| 默认组员 | member / member123 |

## 1. 注册准入控制 (Registration Verification Gating)

### TC-1.1 禁止注册模式
**前置条件**: 管理员已将注册策略设为「禁止注册」。
**步骤**:
1. 浏览器导航至 `http://localhost:5173/login`
2. 点击「注册」Tab
3. 输入用户名 `testuser`、密码 `testpass123`、确认密码 `testpass123`
4. 点击「注册」按钮
**预期结果**: 页面提示「当前不允许自主注册」或类似错误信息，未创建账号。

### TC-1.2 开放注册模式
**前置条件**: 管理员已将注册策略设为「允许直接注册」。
**步骤**:
1. 浏览器导航至 `http://localhost:5173/login`
2. 点击「注册」Tab
3. 输入用户名 `openuser`、密码 `openpass123`、确认密码 `openpass123`
4. 点击「注册」按钮
**预期结果**: 注册成功，自动登录并跳转至工作台。

### TC-1.3 QQ群验证注册模式
**前置条件**: 管理员已将注册策略设为「QQ群验证」，并配置了QQ群号 `123456789`。
**步骤**:
1. 浏览器导航至 `http://localhost:5173/login`
2. 点击「注册」Tab
3. 输入用户名 `qquser`、密码 `qqpass123`、确认密码 `qqpass123`、QQ号 `987654321`
4. 选择资格标签「翻译」
5. 点击「注册」按钮
**预期结果**: 页面显示待验证状态，展示QQ群号 `123456789` 和 `/verify <随机8位码>` 指令，提供一键复制按钮。

### TC-1.4 待验证账号登录拦截
**前置条件**: TC-1.3 已执行，账号处于 `pending_verification` 状态。
**步骤**:
1. 浏览器导航至 `http://localhost:5173/login`
2. 输入用户名 `qquser`、密码 `qqpass123`
3. 点击「登录」按钮
**预期结果**: 不进入系统，页面显示待验证提示，包含QQ群号和 `/verify` 指令。

### TC-1.5 QQ群验证成功激活
**前置条件**: TC-1.3 已执行。
**步骤**:
1. 通过 NoneBot 向配置的QQ群 `123456789` 发送消息 `/verify <8位验证码>`
2. 后端收到事件并处理
3. 浏览器再次执行 TC-1.4 的登录步骤
**预期结果**: 登录成功，账号状态变为 `active`。

### TC-1.6 错误QQ群验证拒绝
**前置条件**: TC-1.3 已执行。
**步骤**:
1. 通过 NoneBot 向非配置QQ群（如 `999999999`）发送 `/verify <8位验证码>`
**预期结果**: 后端拒绝验证，账号保持 `pending_verification`。

## 3. 项目模板管理 (Project Templates)

### TC-3.1 查看模板列表
**前置条件**: 已登录（admin 或 supervisor 角色）。
**步骤**:
1. 导航至 `http://localhost:5173/templates`
**预期结果**: 页面加载模板卡片列表，每张卡片显示模板名称、类型（番剧/电影/合集）、已使用次数、角色链预览。

### TC-3.2 新建项目模板
**前置条件**: 已登录且角色为 supervisor 或 admin。
**步骤**:
1. 进入「项目模板」页面
2. 点击「新建模板」按钮
3. 在弹窗中输入模板名称 `测试番剧模板`

5. 在角色配置中启用「片源」「时轴」「翻译」「后期」「压制」「发布」
6. 将翻译角色的「分配策略」设为「开放认领」，「槽位数」设为 `3`
7. 在「成品配置」中设置分辨率 `1920x1080`、码率 `8000k`、编码器 `x264`
8. 在「交付清单」中添加「测试交付项」
9. 点击「保存」
**预期结果**: 弹窗关闭，模板列表中出现「测试番剧模板」卡片。

### TC-3.3 编辑模板
**前置条件**: TC-3.2 已执行。
**步骤**:
1. 在模板列表中找到「测试番剧模板」卡片
2. 点击卡片上的「编辑」按钮（铅笔图标）
3. 修改模板描述为 `已修改的描述`
4. 点击「保存」
**预期结果**: 弹窗关闭，卡片描述更新。

### TC-3.4 删除模板
**前置条件**: TC-3.2 已执行。
**步骤**:
1. 在模板列表中找到「测试番剧模板」卡片
2. 点击「删除」按钮（垃圾桶图标）
3. 在确认弹窗中点击「确认删除」
**预期结果**: 模板卡片从列表中消失。

### TC-3.5 设为默认模板
**前置条件**: TC-3.2 已执行。
**步骤**:
1. 在模板列表中找到「测试番剧模板」卡片

## 5. 任务工作流 (Task Workflow)

### TC-5.1 查看任务看板
**前置条件**: TC-4.1 已执行。
**步骤**:
1. 进入「测试项目」详情页
2. 默认显示「任务」Tab（Kanban 看板）
**预期结果**: 看板分为「待处理」「进行中」「待审核」「已完成」四列，任务卡片按状态分布在对应列中。

### TC-5.2 指派任务
**前置条件**: TC-4.1 已执行，admin/supervisor 登录。
**步骤**:
1. 进入项目详情页「任务」Tab
2. 在「待处理」列中找到「片源」任务
3. 点击任务卡片
4. 在任务详情弹窗中点击「指派成员」
5. 选择成员 `member`
6. 点击「确认指派」
**预期结果**: 任务卡片移动到「进行中」列（或保持原地但显示 assignee），任务状态变为 `assigned`。

### TC-5.3 开放认领任务
**前置条件**: TC-4.1 已执行，存在翻译任务且策略为开放认领。
**步骤**:
1. 使用 member 账号登录
2. 进入「测试项目」详情页
3. 在「待处理」列中找到翻译任务
4. 点击「认领」按钮
**预期结果**: 任务被 member 认领，状态变为 `in_progress`，看板中任务移动到「进行中」列。

### TC-5.4 分段翻译认领
**前置条件**: TC-4.1 已执行，翻译任务配置了分段认领。
**步骤**:
1. 进入项目详情页
2. 点击翻译任务卡片
3. 在任务详情中点击「选择时间段」
4. 拖动时间轴选择一段范围（如 0:00-5:00）
5. 点击「确认认领」
**预期结果**: 系统记录该时间段已被认领，member 的时间范围内显示为已占用。

### TC-5.5 提交任务成果
**前置条件**: TC-5.3 已执行。
**步骤**:
1. member 进入已认领的翻译任务详情
2. 点击「提交成果」按钮
3. 上传 ASS 字幕文件
4. 输入提交说明 `初稿完成`
5. 点击「确认提交」
**预期结果**: 任务状态变为 `submitted`，任务卡片移动到「待审核」列。

### TC-5.6 审核通过任务
**前置条件**: TC-5.5 已执行。
**步骤**:
1. 使用 admin/supervisor 账号登录
2. 进入项目详情页
3. 在「待审核」列中找到该翻译任务
4. 点击任务卡片
5. 点击「通过」按钮
6. 输入审核意见 `通过，质量很好`
7. 点击「确认」
**预期结果**: 任务状态变为 `review_approved`，卡片移动到「已完成」列，下游任务（如后期）解锁变为 `claimable`。

### TC-5.7 审核驳回任务
**前置条件**: TC-5.5 已执行。
**步骤**:
1. supervisor 进入待审核任务详情
2. 点击「驳回」按钮
3. 输入驳回原因 `时间轴有偏差，请修正`
4. 点击「确认」
**预期结果**: 任务状态变回 `in_progress`，卡片移回「进行中」列，assignee 收到通知。

### TC-5.8 主动退回任务
**前置条件**: TC-5.3 已执行。
**步骤**:
1. member 进入已认领的任务详情
2. 点击「退回任务」按钮
3. 在确认弹窗中点击「确认退回」
**预期结果**: 任务状态变为 `claimable`，卡片移回「待处理」列，审计日志记录退回事件。

### TC-5.9 取消任务冻结下游
**前置条件**: 项目中存在多个串行任务，下游任务尚未开始。
**步骤**:
1. supervisor 进入上游任务详情
2. 点击「取消任务」按钮
3. 确认取消
**预期结果**: 上游任务状态变更，未开始的下游任务被冻结（状态变为 `blocked` 或不可认领）。

### TC-5.10 下游级联重置
**前置条件**: 项目中存在已完成的下游任务。
**步骤**:
1. supervisor 修改上游已完成的任务（如翻译），将其重置为 `in_progress`
2. 确认重置
**预期结果**: 所有下游已完成/已提交的任务自动重置为 `in_progress`，下游 assignee 收到通知。

## 附录：浏览器自动化执行建议

本测试计划建议使用 [Playwright](https://playwright.dev/) 进行浏览器自动化测试。每个测试用例可映射为一个 `test()` 块：

```typescript
// 示例：TC-2.1 管理员登录
import { test, expect } from '@playwright/test';

test('TC-2.1 管理员登录', async ({ page }) => {
  await page.goto('http://localhost:5173/login');
  await page.fill('[name="username"]', 'admin');
  await page.fill('[name="password"]', 'admin123');
  await page.click('button:has-text("登录")');
  await expect(page).toHaveURL(/\/dashboard/);
  await expect(page.locator('text=admin')).toBeVisible();
});
```

对于需要多账号协作的场景（如任务指派与认领），建议按顺序在同一个 worker 中执行，或在 `test.describe.serial` 中组织相关用例。
